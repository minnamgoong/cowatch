
# 1. 개요
이 프로젝트는 폐쇄망 금융권 Core시스템을 수십명 ~ 수백명이 동시에 개발할때, Git의 문제점인 '나중에 충돌 해결'을 미리 파악하고 개선하기 위한 알림 시스템을 만드는 프로젝트 이다.

# 2. 배경
## 2.1. 현재 상황
현재 금융권 Core시스템은 수십명 ~ 수백명이 동시에 개발하고 있다. (약 100여명)
이때 GitLab을 사용하는데, Git은 '나중에 충돌 해결'을 미리 파악하고 개선하기 위한 알림 시스템이 없다.
현재의 Git-Flow의 문제로 인해서, A 기능 브랜치와 B 기능 브랜치가 같은 부분을 순차적으로 수정하고 커밋/푸쉬 한 후, 검증계를 위한 test 브랜치에 머지를 하게 된다.
이때, 배포 일정이 다른 여러개의 기능 브랜치들이 test 브랜치에 머지되고, test 브랜치가 main 브랜치에 머지되는게 아니라 기능 브랜치가 sanity 브랜치에 머지되고, sanity 브랜치가 main 브랜치에 머지된다.

충돌이 일어나는 지점은 다음과 같다
- 기능 브랜치를 test 브랜치에 머지할때
- 기능 브랜치를 sanity 브랜치에 머지할때

기능 브랜치에 푸시가 되었을때, 동시 개발중인 브랜치가 있는지 확인해서 충돌 여부를 확인해줘야 한다.


## 2.2. 문제점
- 기능 브랜치가 sanity 또는 main 브랜치에 머지된 이후에도, 기능 브랜치는 삭제가 되지 않는다.
따라서 기능 브랜치가 현재 '활성 상태'가 아님을 표시하는 flag가 필요하다.


# 3 필요 기능
충돌이 예상되는 기능 브랜치를 발견하면, GitLab의 Issue나 해당 commit에 comment를 작성하여, 개발자가 인지 가능하도록 해야 한다. (Chrome Extension인 Git Notify 이용)
이때, 개발자가 명시적으로 이 충돌을 확인할 수 있는 장치도 해야한다.
